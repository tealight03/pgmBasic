• 폭포수 기법

    소프트웨어 개발 모델 중 하나로, 한 번 떨어지면 거슬러 올라갈 수 없는 폭포수처럼 소프트웨어 개발도 각 단계를 확실히 매듭짓고 다음 단계로 넘어간다는 의미에서 붙여진 이름이다.
    한 단계가 완전히 마무리되면 다음 단계로 넘어가기 때문에 각 단계가 종료될 때마다 결과물이 분명하게 드러난다는 특징을 갖는다.
    하지만 이런 모델에는 큰 문제점이 하나 있는데, 단계가 종료되면 이전 단계를 절대 다시 수행하지 않고 오로지 진행만 하기 때문에
    클라이언트가 이전 단계에 대해 의견을 제시해도 그 의견이 반영되지 않아 최종 결과물을 확인했을 때 고객의 요구사항과 실제 결과물이 많이 다를 수 있다.
    따라서 그때그때 새롭게 요구되는 사항에 대해서 유연하게 대처할 수 없는 방식이므로 리스크가 크다는 점에 유의해야 한다.

  • 스위치와 라우터

    네트워크를 연결할 때 사용되는 기기 종류이다. 허브, 스위치, 라우터 등의 네트워크 연결 장비가 있는데,
    허브는 자신과 연결된 기기 중 어느 하나가 메시지를 다른 기기로 전송하면 그 메시지를 전송하고(방향성 개념X),
    스위치는 동일한 경우에 해당 기기가 메시지를 전송하려고 한 목적지를 판별해 목적지 기기에만 메시지를 전송할 수 있지만 다른 대역대의 기기에는 전송할 수 없고,
    라우터는 동일한 경우에 목적지를 판별해 목적지 기기에만 메시지를 전송할 수도 있고, 다른 대역대의 기기를 서로 연결해주고 있어
    대역대가 다른 기기가 통신하는 것을 가능하게 해준다. 참고로 허브는 해커들이 가장 해킹하기 쉬운 네트워크 장비이다.

  • Dev-ops

    DEV(개발) - OPS(운영)을 통합한 단어로, 개발팀과 운영팀 간의 협력을 통해 소프트웨어 제공 프로세스를 개선하는 방법론을 의미한다.
    2008년 애자일 컨퍼런스에서 앤드루 클레이 쉐이퍼(Andrew Clay Shafer)와 패트릭 드부와(Patrick Debois)가 "애자일 인프라스트럭처"에 대해 논의하며 처음으로 사용되었다.
    주로 소프트웨어 제품이나 서비스를 알맞은 시기에 출시하기 위해서 개발과 운영이 상호의존적으로 대응해야 한다는 의미로 통용된다.
    데브옵스는 실제 결과를 기반으로 일정 주기마다 지속적으로 고객의 요구 사항을 파악하여 즉시 수정사항을 적용한다는 점에서 애자일 기법과 지속적 CI/CD 개념과도 관련 있다.

  • IP

    "Internet Protocol"의 약자로, 네트워크 상의 집주소, 번지수 개념을 일컫는다.
    우리가 일상에서 목적지로 향하기 위해 목적지의 주소를 알면 갈 수 있듯이,
    네트워크에서도 마찬가지로 네트워크 상의 주소를 알면 목적지로 가거나 데이터를 전송하는 등의 명령을 실행할 수 있다.
    IP 주소의 표현 방식은 최대 12개의 숫자, 32 비트로 이루어진 IPv4 방식으로 0.0.0.0 ~ 255.255.255.255 범위의 IP를 사용할 수 있는데,
    전자 통신 기기는 IP를 부여받으면 네트워크 통신이 가능해진다. 시간이 지나면서 태블릿, 웨어러블 PC(애플 워치, 갤럭시 워치 등) 등의 기기가 개발됨으로 인해
    IP를 부여해야하는 기기가 너무 많아짐으로 인해 서브네팅 등의 기술로 대역대를 더 세분화해 IP를 부여하는 방법인 Non-Classful 방식을 이용해서 IP를 부여하기도 한다.
    시간이 지날 수록 관리할 IP가 더욱 더 늘어날 것이기 때문에 최대 32개의 숫자, 128 비트로 이루어진 IPv6 방식을 이용해 사용할 수 있는 IP 개수를 늘릴 예정이라고 한다.

  • HTTP

    "HyperText Transfer Protocol"의 약자로, 웹에서 데이터를 주고받기 위해 사용되는 프로토콜이다.
    웹 브라우저와 웹 서버 사이의 통신을 담당하며, 클라이언트가 서버에 요청(Request)을 보내고, 서버는 클라이언트에게 응답(Response)을 반환하는 방식으로 동작한다.
    클라이언트는 다양한 메서드를 이용해 서버에 원하는 작업을 요구하는데, 메서드에는 GET(리소스 조회), POST(리소스 생성), PUT(리소스 업데이트), DELETE(리소스 삭제)가 있다.
    클라이언트가 서버에게 요청을 하면 서버는 클라이언트의 요청에 대한 처리 결과를 성공 여부 및 처리 상태를 나타내는 숫자로 구성된 상태 코드를 반환한다.
    우리가 가끔씩 웹페이지 오류로 마주하게 되는 404 not found도 http의 상태 코드의 종류 중 하나이다.
    HTTP는 RESTf API같은 다양한 웹 서비스와의 통신에도 활용되는 웹 서비스 프로토콜의 기반이 되는 개념이므로 반드시 숙지해야한다.

  • DHCP

    "Dynamic Host Configuration Protocol"의 약자로 컴퓨터 네트워크에서 IP 주소 및 기타 네트워크 구성 정보를 동적으로 할당하는 프로토콜이다.
    DHCP를 사용하면 네트워크 장치들이 자동으로 IP 주소, 서브넷 마스크, 기본 게이트웨이, DNS 서버 등의 구성 정보를 얻을 수 있다.
    IP 주소 할당, 네트워크 구성 정보 제공, IP 주소 재사용, DHCP 서버와 클라이언트 간의 통신 등의 작업을 DHCP를 통해 자동으로 할 수 있고,
    네트워크 관리의 편의성을 제공하고, IP 주소 관리를 자동화하여 오류를 방지한다는 점 때문에 주로 대규모 네트워크 환경에서 사용된다.
    마이크로소프트 사의 기술이라는 것도 참고해두면 좋다.

  • DNS

    DNS(Domain Name System 혹은 Domain Name Service)는 컴퓨터 네트워크에서 도메인 이름을 IP 주소로 변환하는 시스템이다.
    네트워크에서 통신하기 위해 컴퓨터는 IP 주소를 사용하지만, 사람들은 기억하기 쉬운 도메인 이름을 사용한다.
    이럴 때 DNS는 도메인 이름과 IP 주소 간의 매핑을 관리하여 사용자가 도메인 이름으로 리소스를 찾을 수 있도록 해준다.
    DNS 덕분에 우리는 브라우저에서 사이트를 검색할 때 매번 사이트의 IP를 타이핑하지 않고
    www.naver.com 등의 url, 도메인 이름만으로도 사이트를 검색해 접속할 수 있다.
