• UEFI

    "Unified Extensible Firmware Interface"의 약자로, 컴퓨터의 부트 프로세스를 제어하는 소프트웨어 인터페이스이다.
    보통 "UEFI" 혹은 "UEFI BIOS"라고도 불리며, 대부분의 최신 컴퓨터 시스템에서 사용된다.
    UEFI는 기존의 BIOS(Basic Input/Output System)를 대체하여 컴퓨터 부팅 과정에서 하드웨어 초기화, 부트로더 실행, 운영체제 로딩 등의 작업을 담당한다.
    BIOS와 달리 그래픽 사용자 인터페이스(GUI)를 제공하며, 터치 스크린, 마우스, 네트워크 기능 등과 같은 고급 기능을 지원하고
    보다 큰 부트 디스크 용량, 향상된 보안 기능, 운영체제 독립성 등의 장점을 가지고 있다.

  • WOL

    "Wake-on-LAN"의 약자로, 네트워크를 이용해 원격으로 컴퓨터를 부팅하는 방법을 말한다. 컴퓨터의 전원이 꺼진 상태에서도 가능하다.
    WOL을 사용하려면 컴퓨터의 메인보드나 네트워크 어댑터에서 WOL을 지원해야 하고, 컴퓨터의 BIOS 설정에서 WOL 기능을 활성화해야한다.
    WOL을 사용하면 원격으로 WOL 패킷을 생성하여 대상 컴퓨터의 MAC 주소를 포함한 특정 네트워크 패킷을 로컬 네트워크 또는 인터넷을 통해 전송한다.
    컴퓨터의 네트워크 어댑터는 전원이 꺼져 있어도 WOL 패킷을 수신하고, 이를 인식하여 컴퓨터를 부팅하는 것이다.
    WOL은 회사에서 집에 있는 컴퓨터에 원격으로 접속하여 작업을 해야 할 경우, 서버 관리나 IT 관리 등 원격에서 컴퓨터를 켜고 작업을 수행해야 할 때 유용하다.

  • AC부팅

    AC 부팅은 대부분의 데스크탑 컴퓨터와 일부 서버에서 사용되는 일반적인 부팅 방식이다.
    AC는 Alternating Current(교류 전류)의 줄임말로, 일반적으로 가정이나 사무실에서 사용되는 전원이다.
    일반적으로 컴퓨터를 부팅하는 방식은 AC 부팅 방식이다.
    AC 부팅은 컴퓨터의 전원이 완전히 끊어진 상태에서부터 시작된다.
    전원이 연결되면 메인보드에 전기가 공급되고, 메인보드는 초기 부팅 프로세스를 시작하는데,
    이 프로세스에서는 하드웨어 초기화, BIOS 또는 UEFI 로드, 부트 로더 실행 등의 단계를 거쳐 운영체제가 시작된다.
    참고로 몇몇 모바일 기기는 배터리 전원을 사용하여 부팅할 수도 있으며, 이를 "배터리 부팅"이라고 한다.

  • 가상머신(네이티브)와 도커

    가상머신(Virtual Machine)은 현재 사용하는 운영체제 내에서 다른 운영체제를 동작할 수 있도록 해주는 응용 소프트웨어를 말한다.
    "가상"이라는 이름에서 알 수 있듯이 실제 존재하지는 않지만 가상의 새로운 컴퓨터 하드웨어가 존재하는 것처럼 여겨
    그 하드웨어에 Windows, MacOS, Linux 등의 현재 Host컴퓨터의 OS와 같거나 다른 OS들을 설치해서 그 컴퓨터를 현실에서 동작하는 것과 같이 사용할 수 있다.
    가상머신 중 네이티브(Native)는 주어진 플랫폼 또는 운영체제에서 직접 실행되는 응용 프로그램을 말한다.
    하지만 가상머신에는 몇 가지 단점이 존재하는데, 그 중 대표적인 것은 가상머신의 OS를 현재 Host컴퓨터에서 동작하고, 그 내부 동작들도 Host컴퓨터가 처리하게 되므로
    경우에 따라 동작 시간이 오래 걸리기도 하고, Host컴퓨터가 처리해야할 명령이 너무 많아진다는 것이다.
    이를 처리하기 위해 "도커"라는 컨테이너화(Containerization) 기술을 제공하는 오픈 소스 플랫폼을 사용한다.
    컨테이너화 기술은 "컨테이너"라는 고립된 공간 내에서 실행되고 해당 컨테이너에 필요한 라이브러리, 실행 파일, 구성 파일 등의 종속성이 포함되어 있는 컨테이너를 이용해
    호스트 시스템의 운영체제 커널을 공유하는 기술을 말하는데, 이를 이용하면 가상머신보다 가볍고 빠르게 시스템 리소스를 활용할 수 있다.
    도커를 이용하면 응용 프로그램과 그에 필요한 종속성을 격리된 환경인 컨테이너에 패키징할 수 있다.

  • 라즈베리파이

    라즈베리 파이(Raspberry Pi)는 싱글 보드 컴퓨터로, 영국의 라즈베리 파이 재단에서 개발한 소형 컴퓨터이다.
    라즈베리 파이는 작고 저렴한 가격으로 구매할 수 있고, 보통 크레딧 카드 크기의 보드에 CPU, 메모리, 그래픽, I/O 포트 등이 통합된 구성으로 제작된다.
    다양한 모델과 사양이 있는데 최신 모델은 ARM 기반 프로세서를 사용하고, Raspbian, Ubuntu 등 여러 가지 운영체제를 지원한다.
    주로 교육, IoT 프로젝트, 서버 컴퓨터 등으로 사용된다.

  • VPN

    VPN(Virtual Private Network)은 공용 네트워크를 통해 안전하게 데이터를 전송하기 위한 보안 기술이다.
    VPN은 인터넷 연결을 이용해 사용자의 장치와 원격 서버 또는 네트워크 간에 암호화된 터널을 생성하여 데이터의 기밀성과 안전성을 보장한다.
    주로 개인 정보 보호, 네트워크 우회 접근, 원격 액세스 등을 위해 사용한다.

  • RDP정의와 공격

    RDP(Remote Desktop Protocol)는 원격 컴퓨터와의 연결을 위한 프로토콜이다.
    RDP 기술은 Windows 운영체제에서 원격 데스크톱 기능을 제공하고, 사용자가 원격으로 다른 컴퓨터에 접속하여 해당 컴퓨터를 제어할 수 있게 한다.
    RDP를 사용하면 원격으로 작업하는 동안 원격 컴퓨터의 화면, 파일 및 애플리케이션을 실시간으로 확인하고 조작할 수 있다.
    RDP를 사용하면 편리하게 원격 접속을 할 수 있지만, 해커들이 RDP를 악용해 공격을 해올 수도 있다.
    공격의 유형에는 RDP 서버에 로그인하기 위해 로그인이 성공할 때까지 모든 경우를 대입하는 브루트 포스 공격(무차별 대입 공격),
    RDP 소프트웨어 또는 관련 서비스의 취약점을 이용하여 공격자가 원격 컴퓨터에 액세스하고 시스템을 침해하는 취약점 악용 공격(Exploitation of Vulnerabilities),
    네트워크 트래픽을 가로채어 원격 접속 시도를 중간에서 감시하고 조작해 사용자의 RDP 세션을 도청하거나 조작하는 중간자 공격(Man-in-the-Middle Attack) 등이 있다.
    이런 공격을 방어하기 위해서는 최신 보안 업데이트 적용, 네트워크 방화벽 구성, RDP 서버 접속 시도 기록 확인을 평소에 하는 습관을 들이는 것이 좋다.

  • 타입스크립트와 자바스크립트

    타입스크립트(typescript)와 자바스크립트(javascript)는 둘 다 웹 개발에 사용되는 프로그래밍 언어이다.
    자바스크립트는 HTML, CSS와 함께 웹 개발, 프론트엔드의 대표적인 언어로 꼽힌다.
    웹 페이지 내의 다이나믹한 부분을 구현하는데 주로 사용되고 타입이 유동적이고 약한 것이 특징이다.
    예를 들어, "1" + 0이라는 연산을 명령하면 "10", 선행되는 데이터 타입인 문자열 타입으로 결과가 도출된다.
    이런 특성 때문에 자바스크립트는 특정 상황에서 예상치 못한 에러가 발생할 수 있다.
    이와 반대로, 타입스크립트는 타입이 강하다는 특성을 갖는다.
    때문에 위에서 제시한 예시와 같은 명령어를 작성할 경우 바로 에러가 난다.
    이런 이유로 타입스크립트를 사용하는 프로그래머들이 점점 증가하는 추세를 보이고 있다.

  • nest-JS

    NestJS는 Node.js를 위한 프레임워크로, 서버 사이드 애플리케이션을 구축하기 위한 도구와 구조를 제공합니다.
    Angular의 영향을 받아 Angular와 비슷한 구문과 패턴을 사용하며, 확장성과 모듈화를 강조하는 프레임워크이다.
    TypeScript, 미들웨어, WebSocket을 지원하고 모듈 구조를 사용한다는 특징이 있다.
    서버 사이드 애플리케이션부터 마이크로서비스 아키텍처까지 다양한 유형의 애플리케이션을 개발할 수 있도록 지원하고,
    생산성과 유지보수성을 높이기 위해 다양한 기능과 패턴을 제공한다.

  • CI/CD

    소프트웨어의 통합과 배포에 관련한 개념이다.
    CI(Continus Integration)은 지속적인 통합, 즉 "빌드와 테스트의 자동화"를 의미한다.
    개발의 모든 과정을 완료한 후 소프트웨어의 품질을 검토&관리하던 고전적인 방식의 단점을 해소하기 위한 개념으로,
    말 그대로 코드의 변경 사항이 주기적으로 빌드 및 테스트되어 공유 리포지토리에 통합되는 과정을 통해 품질을 유지하며 개발을 진행하는 방법을 말한다.
    CD(Continus Delivery & Continus Deployment)는 CI의 연장선 개념으로, CI 과정을 통과한 버전을 마지막에 배포하는 과정에 대한 개념이다.
    코드 변경 사항이 CI 과정을 모두 통과하면 프로덕션에 자동으로 배포되는 되기 때문에 신속하고 능률적으로 사용자에게 새로운 기능을 제공할 수 있다.
    CI/CD 파이프라인을 사용하면 프로젝트의 프로세스를 명확히 할 수 있고, 코드의 품질이 높아지고, 버전 관리를 효율적으로 할 수 있어서 개발 민첩성이 향상된다는 장점이 있다.

  • 에자일 기법

    애자일은 신속한 반복 작업을 통해 실제 작동 가능한 소프트웨어를 개발하여 지속적으로 제공하기 위한 소프트웨어 개발 방식을 말한다.
    소프트웨어 개발을 위한 유연하고 협력적인 방법론으로, 프로젝트를 작은 주기로 반복하며 요구사항의 변경에 적응하고 클라이언트와의 긴밀한 협력을 강조한다.
    적응적인 계획, 작은 팀의 자율성, 지속적인 품질 향상을 특징으로 하며, 개발 프로세스의 유연성과 효율성을 높여 가치 있는 소프트웨어를 제공한다.
