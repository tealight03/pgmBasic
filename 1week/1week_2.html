• 폭포수 기법

    소프트웨어 개발 모델 중 하나로, 한 번 떨어지면 거슬러 올라갈 수 없는 폭포수처럼 소프트웨어 개발도 각 단계를 확실히 매듭짓고 다음 단계로 넘어간다는 의미에서 붙여진 이름이다.
    한 단계가 완전히 마무리되면 다음 단계로 넘어가기 때문에 각 단계가 종료될 때마다 결과물이 분명하게 드러난다는 특징을 갖는다.
    하지만 이런 모델에는 큰 문제점이 하나 있는데, 단계가 종료되면 이전 단계를 절대 다시 수행하지 않고 오로지 진행만 하기 때문에
    클라이언트가 이전 단계에 대해 의견을 제시해도 그 의견이 반영되지 않아 최종 결과물을 확인했을 때 고객의 요구사항과 실제 결과물이 많이 다를 수 있다.
    따라서 그때그때 새롭게 요구되는 사항에 대해서 유연하게 대처할 수 없는 방식이므로 리스크가 크다는 점에 유의해야 한다.

  • 스위치와 라우터

    네트워크를 연결할 때 사용되는 기기 종류이다. 허브, 스위치, 라우터 등의 네트워크 연결 장비가 있는데,
    허브는 자신과 연결된 기기 중 어느 하나가 메시지를 다른 기기로 전송하면 그 메시지를 전송하고(방향성 개념X),
    스위치는 동일한 경우에 해당 기기가 메시지를 전송하려고 한 목적지를 판별해 목적지 기기에만 메시지를 전송할 수 있지만 다른 대역대의 기기에는 전송할 수 없고,
    라우터는 동일한 경우에 목적지를 판별해 목적지 기기에만 메시지를 전송할 수도 있고, 다른 대역대의 기기를 서로 연결해주고 있어
    대역대가 다른 기기가 통신하는 것을 가능하게 해준다. 참고로 허브는 해커들이 가장 해킹하기 쉬운 네트워크 장비이다.

  • 클라우드 서버

    클라우드 서버는 애플리케이션을 제공하고, 정보를 처리하거나 데이터 스토리지를 제공하는 강력한 물리적 또는 가상 인프라이다.
    클라우드 서비스 제공업체는 IaaS(Infrastructure-as-a-service) 모델을 사용하여 고객이 사용할 수 있는 가상 또는 물리적 서버를 구축한다.
    가상화된 환경에서 실행되며, 필요에 따라 자원을 동적으로 할당하거나 조정할 수 있고, 클라우드 서비스 제공 업체가 관리하므로
    사용자는 자원 관리, 보안 패치, 유지 보수 등의 부담을 덜 수 있다. 클라우드 서버는 확장성, 가용성, 유연성 및 비용 효율성을 제공하여 대부분의 사용자에게 유용한 방법으로 사용될 수 있다.
    참고로 몇몇 클라우드 서버는 하나의 물리적 서버를 여러 가상 서버로 분할하는 가상화 소프트웨어를 사용하여 구축된다. [IBM 웹사이트에서 추출]

  • nginx

    웹 서버용 오픈 소스 소프트웨어로, 러시아의 프로그램 개발자인 이고르 시쇼브(Igor Sysoev)가 개발하여 2004년 발표하였다.
    nginx(이하 엔진엑스)가 제공하는 각 기능은 모듈화되어 있어 효율적인 운영을 할 수 있고,
    흔히 사용되는 아파치(Apache) 웹 서버 소프트웨어보다 메모리 사용이 효율적이며 클라이언트 처리 속도가 더 빠르다.
    아파치(Apache)는 사용자 트래픽이 폭증하는 경우 신속히 처리하지 못하는 문제가 있는 반면,
    엔진엑스는 아파치의 문제점을 보완해 메모리를 사용량이 적어 비교적 가볍고 대량 접속 문제를 신속히 효율적으로 처리한다.
    엔진엑스는 기본적인 웹 서버 기능 외에 추가로 ‘캐싱(caching)’, ‘트래픽 부하 균형(load balancing)’, ‘TLS/SSL 보안’, ‘메일 프록시’ 등의 기능을 제공한다.

  • 메모리 변조 해킹-Rust의 메모리 프로텍션

    메모리 변조 해킹은 해커가 실행 중인 프로그램의 메모리를 수정하여 제어 권한을 얻는 공격이다.
    메모리 변조 해킹 공격을 방지하기 위해 Rust 같은 언어는 메모리 프로텍션 메커니즘을 제공한다.
    메모리 안정성을 강조하는 Rust언어는 변수에 대한 소유권은 단일 스레드나 여러 스레드 간에 안전하게 전달되며,
    데이터 경쟁이나 잘못된 메모리 접근으로부터 보호할 수 있는 "소유권 시스템", 메모리 접근을 범위 검사하여
    버퍼 오버플로우와 같은 메모리 침범을 방지하는 "범위 검사", Mutex, Arc, RwLock 등의 타입을 사용하여 동시성 문제를 방지하고,
    데이터 레이스 및 다른 스레드 간의 메모리 침범을 방지하기 위해 안전한 스레드 동기화 기능을 제공한다.
    메모리 프로텍션만으로는 완벽한 보안을 장담할 수 없으므로 외부 라이브러리와의 상호 작용, 안전하지 않은 코드 연결에 대해서도 신중하게 처리해야 한다.

  • Dev-ops

    DEV(개발) - OPS(운영)을 통합한 단어로, 개발팀과 운영팀 간의 협력을 통해 소프트웨어 제공 프로세스를 개선하는 방법론을 의미한다.
    2008년 애자일 컨퍼런스에서 앤드루 클레이 쉐이퍼(Andrew Clay Shafer)와 패트릭 드부와(Patrick Debois)가 "애자일 인프라스트럭처"에 대해 논의하며 처음으로 사용되었다.
    주로 소프트웨어 제품이나 서비스를 알맞은 시기에 출시하기 위해서 개발과 운영이 상호의존적으로 대응해야 한다는 의미로 통용된다.
    데브옵스는 실제 결과를 기반으로 일정 주기마다 지속적으로 고객의 요구 사항을 파악하여 즉시 수정사항을 적용한다는 점에서 애자일 기법과 지속적 CI/CD 개념과도 관련 있다.

  • 포트포워딩(프록시)

    포트 포워딩은 네트워크 트래픽을 외부 네트워크와 내부 네트워크 사이에서 전달하기 위해 특정 포트를 연결하는 것을 의미합니다.
    일반적으로 라우터나 방화벽에서 사용되며, 내부 네트워크에 있는 장치나 서비스에 외부에서 접근할 수 있도록 한다.
    예를 들어 외부 네트워크에서 특정 포트로 요청이 들어오면 라우터나 방화벽이 해당 요청을 내부망의 특정 IP 주소와 포트로 전달해서
    내부 서버가 해당 포트로 전달된 요청을 수행해서 응답을 반환하고, 이 응답은 다시 라우터(방화벽)을 통해 외부 네트워크로 전달 된다.
    주로 원격으로 내부 서버에 접속하거나, 웹 서비스를 외부에서 호스팅하는 경우에 사용할 수 있습니다.
    프록시 서버는 클라이언트와 서버 간의 중계 역할을 하는 서버로, 클라이언트가 서버에 직접 연결하는 대신 클라이언트의 요청을 받아
    해당 요청을 서버로 전달하고, 서버의 응답을 다시 클라이언트로 전달하는데 이러한 중계 과정에서
    프록시는 클라이언트와 서버 간의 네트워크 통신을 제어하고 필터링하는 역할을 한다.

  • 리버스 엔지니어링

    완성된 제품을 분석하여 제품의 기본적인 설계 개념과 적용 기술을 파악하고 재현하는 소프트웨어의 한 세부 분야이다.
    제품 설계 -> 개발 작업 -> 제품화의 일반적인 설계 방식을 거꾸로 파고들어 소프트웨어의 실행 파일을 수정하거나 프로그램의 동작을 변경할 수 있고,
    제품의 설계 방식을 분석해 어떤 기술을 쓰면 좀 더 개선된 제품을 만들 수 있을지 연구할 수 있다.
    이를 이용해 유사한 형태의 복제 프로그램 혹은 기능이 향상된 프로그램도 개발할 수 있다. 소프트웨어에 대한 역공학 자체는 위법 행위가 아니지만,
    대부분의 제품이 이에 대해 금지하고 있어 이러한 수법으로 개발한 제품은 지적 재산권을 침해할 위험성이 있다는 점 유의해야 한다.

  • 디지털 포렌식

    일반적으로 우리가 아는 디지털 포렌식은 디지털 디바이스 사용 중 발생한 범죄나 사건과 관련된 디지털 증거를 수집, 분석하는 과학적인 수사법이다.
    PC, 스마트폰, 태블릿 등의 디지털 기기를 조사하여 정보를 추출하고 범죄 혹은 사건에 대한 증거 수집을 위해 기기의 잠금을 풀기도 하고, 삭제된 데이터를 복구해 증거를 수집한다.
    공부하는 것이 많이 어렵고 이 분야로 진로를 잡은 사람들이 그리 많지 않아 관심과 실력이 있다면 디지털 포렌식에 대해 공부하는 것도 매우 좋다.

  • IP

    "Internet Protocol"의 약자로, 네트워크 상의 집주소, 번지수 개념을 일컫는다.
    우리가 일상에서 목적지로 향하기 위해 목적지의 주소를 알면 갈 수 있듯이,
    네트워크에서도 마찬가지로 네트워크 상의 주소를 알면 목적지로 가거나 데이터를 전송하는 등의 명령을 실행할 수 있다.
    IP 주소의 표현 방식은 최대 12개의 숫자, 32 비트로 이루어진 IPv4 방식으로 0.0.0.0 ~ 255.255.255.255 범위의 IP를 사용할 수 있는데,
    전자 통신 기기는 IP를 부여받으면 네트워크 통신이 가능해진다. 시간이 지나면서 태블릿, 웨어러블 PC(애플 워치, 갤럭시 워치 등) 등의 기기가 개발됨으로 인해
    IP를 부여해야하는 기기가 너무 많아짐으로 인해 서브네팅 등의 기술로 대역대를 더 세분화해 IP를 부여하는 방법인 Non-Classful 방식을 이용해서 IP를 부여하기도 한다.
    시간이 지날 수록 관리할 IP가 더욱 더 늘어날 것이기 때문에 최대 32개의 숫자, 128 비트로 이루어진 IPv6 방식을 이용해 사용할 수 있는 IP 개수를 늘릴 예정이라고 한다.

  • TCP와 UDP

    TCP는 "Transmission Control Protocol"의 약자로, 연결을 지향하는 프로토콜이다.
    두 기기가 서로 통신(연결)된 후 데이터를 주고 받고 안전하게 연결을 종료하는 방식이기 때문에 데이터 전송에 다소 시간이 소요된다.
    이와 반대로 UDP는 "User Datagram Protocol"의 약자로, 비연결성 프로토콜이다.
    두 기기가 서로 데이터를 주거나 받기 전 연결을 하지 않고, 일방적으로 데이터를 전송하는 방식의 프로토콜이다.
    TCP는 연속성보다 신뢰성 있는 전송이 중요할 때 사용되는 프로토콜이며,
    UDP는 TCP보다 빠르고 네트워크 부하가 적다는 장점이 있지만, 신뢰성 있는 데이터 전송을 보장하지는 않는다(확인을 안하기 때문).
    그렇기 때문에 신뢰성보다는 연속성이 중요한 실시간 스트리밍과 같은 서비스에 자주 사용된다.

  • HTTP

    "HyperText Transfer Protocol"의 약자로, 웹에서 데이터를 주고받기 위해 사용되는 프로토콜이다.
    웹 브라우저와 웹 서버 사이의 통신을 담당하며, 클라이언트가 서버에 요청(Request)을 보내고, 서버는 클라이언트에게 응답(Response)을 반환하는 방식으로 동작한다.
    클라이언트는 다양한 메서드를 이용해 서버에 원하는 작업을 요구하는데, 메서드에는 GET(리소스 조회), POST(리소스 생성), PUT(리소스 업데이트), DELETE(리소스 삭제)가 있다.
    클라이언트가 서버에게 요청을 하면 서버는 클라이언트의 요청에 대한 처리 결과를 성공 여부 및 처리 상태를 나타내는 숫자로 구성된 상태 코드를 반환한다.
    우리가 가끔씩 웹페이지 오류로 마주하게 되는 404 not found도 http의 상태 코드의 종류 중 하나이다.
    HTTP는 RESTf API같은 다양한 웹 서비스와의 통신에도 활용되는 웹 서비스 프로토콜의 기반이 되는 개념이므로 반드시 숙지해야한다.

  • DHCP

    "Dynamic Host Configuration Protocol"의 약자로 컴퓨터 네트워크에서 IP 주소 및 기타 네트워크 구성 정보를 동적으로 할당하는 프로토콜이다.
    DHCP를 사용하면 네트워크 장치들이 자동으로 IP 주소, 서브넷 마스크, 기본 게이트웨이, DNS 서버 등의 구성 정보를 얻을 수 있다.
    IP 주소 할당, 네트워크 구성 정보 제공, IP 주소 재사용, DHCP 서버와 클라이언트 간의 통신 등의 작업을 DHCP를 통해 자동으로 할 수 있고,
    네트워크 관리의 편의성을 제공하고, IP 주소 관리를 자동화하여 오류를 방지한다는 점 때문에 주로 대규모 네트워크 환경에서 사용된다.
    마이크로소프트 사의 기술이라는 것도 참고해두면 좋다.

  • DNS

    DNS(Domain Name System 혹은 Domain Name Service)는 컴퓨터 네트워크에서 도메인 이름을 IP 주소로 변환하는 시스템이다.
    네트워크에서 통신하기 위해 컴퓨터는 IP 주소를 사용하지만, 사람들은 기억하기 쉬운 도메인 이름을 사용한다.
    이럴 때 DNS는 도메인 이름과 IP 주소 간의 매핑을 관리하여 사용자가 도메인 이름으로 리소스를 찾을 수 있도록 해준다.
    DNS 덕분에 우리는 브라우저에서 사이트를 검색할 때 매번 사이트의 IP를 타이핑하지 않고
    www.naver.com 등의 url, 도메인 이름만으로도 사이트를 검색해 접속할 수 있다.

  • OSI-7계층

    OSI (Open Systems Interconnection) 7 Layer 모델은 표준 프로토콜을 사용하여 다양한 통신 시스템이 통신할 수 있도록 국제표준화기구에서 만든 개념 모델이다.
    이 모델은 네트워크 통신을 물리 계층(Physical Layer), 데이터 링크 계층(Data Link Layer), 네트워크 계층(Network Layer), 전송 계층(Transport Layer),
    세션 계층(Session Layer), 표현 계층(Presentation Layer), 응용 계층(Application Layer)의 단계로 구별하여 관리 가능하도록 설계되었다.
    각각 독립적으로 계층화된 OSI 모델은 표준화된 방식으로 네트워크 프로토콜을 설계하고 구현할 수 있도록 해준다.
    각 계층은 본 계층의 특징을 이용하여 상위 계층과 하위 계층 간의 상호 작용을 지원한다. 네트워크의 기본 토대가 되는 개념이라고 할 수 있다.
